# Prisma & Database Management

This directory contains the database schema, migrations, and seeding logic for the Nestlancer project. We use **Prisma** as our ORM with a modular schema structure and custom seeding strategy.

## 📂 Project Structure

- `schema/`: Modular Prisma schema files. The main entry point is `schema.prisma`.
- `migrations/`: SQL migration files generated by Prisma.
- `seeds/`: Database seeding logic.
  - `index.ts`: Main seeding entry point.
  - `01-09-*.seed.ts`: Core system seeds (Roles, Permissions, etc.).
  - `dev/`: Development-only seeds (Test users, projects, etc.).
- `prisma.config.ts`: Prisma configuration using the `earlyAccess` multi-file schema feature.

## 🛠 Development Commands

All database commands should be run from the root directory using `pnpm`.

### 1. Schema Management

- **Generate Prisma Client**: Updates the Prisma Client after schema changes.
  ```bash
  # Ensure DATABASE_URL is in the environment
  export DATABASE_URL=$(grep DATABASE_URL .env.development | cut -d '=' -f2-) && pnpm db:generate
  ```
- **Create & Apply Migration**: Use this when you change any `.prisma` file in `prisma/schema/`.
  ```bash
  export DATABASE_URL=$(grep DATABASE_URL .env.development | cut -d '=' -f2-) && pnpm db:migrate
  ```

### 2. Seeding & Reset

- **Seed Database**: Runs the seeding scripts in `prisma/seeds/`.
  ```bash
  pnpm db:seed
  ```
- **Reset Database**: Drops all data, re-runs migrations, and re-seeds. **Caution: Destructive!**
  ```bash
  pnpm db:reset
  ```

### 3. Visualization

- **Prisma Studio**: Opens a GUI to explore and edit your database data.
  ```bash
  pnpm db:studio
  ```

## 🚀 Production Commands

In production environments, avoid `migrate dev`. Use the following:

- **Apply Pending Migrations**: Safely applies migrations without creating new ones or resetting.
  ```bash
  pnpm db:migrate:deploy
  ```

## 🧪 Testing Commands

- **Run Docker Test DB**: Spins up a clean PostgreSQL instance for integration tests.
  ```bash
  pnpm docker:test
  ```

## 🔑 Environment Variables

Required variables in your `.env` file:

- `DATABASE_URL`: Main connection string for Prisma.
- `DATABASE_READ_URL`: (Optional) Connection string for read-only replicas.
- `NODE_ENV`: Set to `production` to skip development seeds.

### Loading Environment Variables

Since `prisma.config.ts` requires these variables explicitly, you can load them before running any `pnpm db:*` command:

```bash
export $(grep -v '^#' .env.development | xargs) && pnpm db:generate
```

Alternatively, ensures a `.env` file exists (symlink is created by the system):

```bash
# Check if .env exists
ls -la .env
```

---

_Note: This project uses Prisma 7+ with driver adapters. Direct connections require a pool manager as configured in `prisma/seeds/index.ts`._
